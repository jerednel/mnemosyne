The problem

Every assistant is brilliant. None of them remember.

Today's AI memory is vector search over chat logs — text fragments with no identity, no relationships, and no sense of time. The cost compounds across every assistant your team runs.

Context fragmentation

The same people, systems, and projects get re-explained to every assistant, in every tool, in every session. Knowledge is paid for repeatedly and owned by no one.

No shared cognition

Each vendor keeps its own memory silo. What your coding agent learned this morning is invisible to your research assistant this afternoon — and lost entirely when you switch models.

No sense of time

Retrieval systems overwrite state instead of modeling it. "Who was the account owner in Q2?" is unanswerable when yesterday's truth was deleted to store today's.

Architecture

Two tiers, one graph. The privacy boundary is structural.

Shared knowledge and private context are physically separate stores, merged at query time. Private data reaching the shared tier isn't forbidden by policy — it's impossible by construction.

Tier 1 · Canonical

Hosted shared ontology

Canonical entities, aliases, and relationships served over authenticated HTTP with per-tenant keys. Opened read-only at runtime — no write path exists.

Tier 2 · Overlay

Private, local-first memory

Your entities, relationships, memories, and learned aliases. Extends and shadows the shared graph. Never transmitted, never synced without consent.

Provable isolationAn automated test asserts the shared-tier database is byte-identical after a full private write workload.
Swappable backendsThe canonical tier sits behind a storage protocol — SQLite today, multi-tenant Postgres tomorrow, nothing above changes.
Append-only historyEvery write lands in an immutable assertion log in the same transaction. State and audit trail cannot drift.
Capabilities

Structure-first memory, not retrieval-first.

Identity

Resolution that learns

"DBX", "the lakehouse vendor", and "Databrics" all resolve to one canonical entity. High-confidence matches are learned as aliases; ambiguous ones become reviewable proposals — never silent links.

Time

Facts supersede, never overwrite

Every fact carries a validity window. New truths close old ones and link the succession, so "what was true in July?" is a query, not an apology.

Trust

Per-fact provenance

Which assistant asserted it, in which session, at what confidence — captured from the protocol handshake, never self-reported. Seed data, assistant claims, and inference stay permanently distinguishable.

Privacy

Private overlays on shared entities

Attach your notes, nicknames, and attributes to a canonical entity without duplicating or publishing it. The merged view exists only on your side of the boundary.

Reasoning

Graph queries, not text search

Typed, ontology-validated relationships with direction, depth, and as-of time travel. Assistants query meaning and structure — vector recall is the fallback, not the foundation.

Extensible

Pluggable semantic matching

Layer embedding similarity over lexical matching with one environment variable. Paraphrases resolve where string distance can't reach — no SDK lock-in, no default overhead.

Enterprise readiness

Built for the questions your security review will ask.

✓

Structural data isolation

The shared tier is opened read-only at the storage layer. There is no code path from private memory to shared storage — verified by byte-identity tests.

✓

Authenticated multi-tenant access

Bearer-token auth with per-key identity on every hosted-tier request. Fail-closed startup; every response is attributed for audit and future quotas.

✓

Complete audit trail

An append-only assertion log records every fact, supersession, and merge decision with actor and timestamp — written atomically with the state it describes.

✓

No model or vendor lock-in

The only interface is the open Model Context Protocol. Swap models and assistants freely; the memory graph is the durable asset and it belongs to you.

✓

Open source, permissive stack

MIT licensed with three lightweight permissive dependencies. Auditable end to end; no copyleft exposure, no proprietary runtime.

✓

Memory pollution as a design constraint

Ambiguous identity matches are proposed, not committed. Confidence scores, human review hooks, and provenance keep hallucinated facts out of the graph.

Interoperability

One fabric. Every assistant your team already uses.

Cross-assistant continuity isn't a claim — it's a passing test. Assistant B recalls what assistant A learned, sees exactly who asserted it, and evolves the shared record.
